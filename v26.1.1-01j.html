Bulletbox 26.1.1-01j
Alpha Unreleased
Networking & Infrastructure
  • Thread-Safe Socket System: Fixed the major "Broken pipe" and mass-disconnect issues by implementing synchronized WriterLocks for all outbound network traffic. Multiple networking threads can now safely transmit simultaneously without packet corruption. (26.1.1-01j.1/1)
  • Connected Player Collection Locking: Added synchronization protection to the ConnectedPlayers collection to prevent crashes during joins, disconnects, and player broadcast events. (26.1.1-01j.1/2)
  • Expanded Player Synchronization: The server now synchronizes player rotation and currently held item IDs to nearby clients for accurate combat rendering and visual consistency. (26.1.1-01j.1/3)
Combat & Weapon Systems
  • New Weapon Types: Added three new weapon classes:
    • Sword (S)
    • Kanabo (K)
    • Spear (P)
    (26.1.1-01j.2/1)
  • Weapon Swing Animations: Sword and Kanabo attacks now use smooth 120-degree arc swing animations instead of static rotation snapping. (26.1.1-01j.2/2)
  • Spear Stab Animation: Spears now perform a fast linear thrust animation using sine-wave interpolation for smoother attack motion. (26.1.1-01j.2/3)
  • Dynamic Weapon Depth Sorting: Weapons now render behind the player while facing upward and in front while facing downward, improving 2D directional perspective. (26.1.1-01j.2/4)
  • Hand-Based Weapon Pivoting: Weapon rotation origin now uses a calculated hand position at the player edge instead of rotating directly from the body center. (26.1.1-01j.2/5)
  • Visual Weapon Scaling: Held weapons are now rendered at 2x scale in-world to improve readability and combat impact. (26.1.1-01j.2/6)
Gameplay & Movement
  • Normalized Movement Vectors: Fixed unintended fast diagonal movement by implementing vector normalization. Player movement speed is now consistently capped at 350 units/s regardless of movement direction. (26.1.1-01j.3/1)
  • Off-Screen Player Tracking: Added edge-screen tracking arrows that point toward the two nearest off-screen players to improve combat navigation and player discovery. (26.1.1-01j.3/2)
  • Directional Arrow Rendering: Tracking arrows now render as filled blue indicators with dark-blue borders for improved visibility across all environments. (26.1.1-01j.3/3)
User Interface & Social Systems
  • Chat System: Added a bottom-anchored chat interface where new messages push older messages upward automatically. (26.1.1-01j.4/1)
  • Chat Background Rendering: Chat lines now render with semi-transparent gray backgrounds for improved readability during gameplay. (26.1.1-01j.4/2)
  • Message Fade System: Chat messages now smoothly fade after 13 seconds and fully disappear after 15 seconds. (26.1.1-01j.4/3)
  • Chat Input Blocking: Full gameplay input is now disabled while chat is active (/) to prevent accidental movement and combat actions while typing. (26.1.1-01j.4/4)
  • TAB Player List: Holding TAB now displays up to 30 nearby players in a 3-column by 10-row overlay sorted by proximity. The local player is highlighted in Sky Blue. (26.1.1-01j.4/5)
  • Item Tooltips: Added rounded RPG-style item tooltips displaying:
    • Item Name
    • Damage
    • Speed
    • Range
    • Knockback
    (26.1.1-01j.4/6)
  • Tooltip Boundary Detection: Tooltips now automatically reposition to remain fully visible on-screen and prevent clipping. (26.1.1-01j.4/7)
  • Hotbar Visual Rotation: Hotbar item sprites are now rotated 45 degrees counter-clockwise for a more polished RPG inventory appearance. (26.1.1-01j.4/8)
  • Discord Rich Presence: Added Discord Rich Presence integration, allowing player activity and in-game status to display directly on Discord. (26.1.1-01j.4/9)
Rendering & Technical Refinements
  • Direction Arrow Winding Fix: Corrected triangle winding order for directional arrows to ensure proper filling behavior across graphics backends. (26.1.1-01j.5/1)
  • Color Constructor Ambiguity Fix: Added explicit (byte) casting for Raylib_cs.Color initialization to resolve compiler ambiguity issues. (26.1.1-01j.5/2)
  • Codebase Cleanup: Removed unused visualHealth and lerpSpeed variables from the HealthBar class to reduce build warnings and simplify rendering logic. (26.1.1-01j.5/3)
  • Compiler Stability Fix: Resolved critical nested method definition issues inside Player.cs which caused compiler instability in specific builds. (26.1.1-01j.5/4)
Notes
  • This update includes one of the largest multiplayer infrastructure rewrites completed so far. (26.1.1-01j.6/1)
  • Weapon systems introduced in this update are designed to support future combat mechanics including status effects, directional blocking, and advanced attack chaining. (26.1.1-01j.6/2)
  • Additional social systems and larger multiplayer testing are planned for future updates. (26.1.1-01j.6/3)
