Architecture File Breakdown
  • Program: This is the entry point and the "Brain" of the game. It holds the global state machine (GameState) and the main loop. Its only job is to ask the SaveManager if a user exists at startup and then tell the active screen to update and draw every frame.
  • HomeScreen: This is the primary menu hub. It handles the title, the main buttons (Play, Mods, Options, Quit), and most importantly, it owns the "Static" GameOfLife background. By making the background static here, every other class in the game can access and draw it without needing to create their own copy.
  • UIButton: The base for all interactive text. Instead of using standard boxes, this class calculates a hitbox based on text size. It uses a "Lerp" (Linear Interpolation) formula to smoothly scale the text up and brighten it whenever the mouse is hovering, giving the menu a "liquid" feel.
  • LoginScreen: Handles the credential entry. It uses a character buffer to capture typing and manages "Focus" logic—allowing you to click between the Username and Password fields. It also features conditional rendering to hide placeholder text the moment you start typing.
  • GameOfLife: The background simulation engine. It runs the logic for the cells and uses a Queue to keep a history of the last 3 frame counts. If the "Live Cell" count stays the same for too long (stagnation), it triggers a board reset to keep the background from ever staying still.
  • SaveManager: The I/O utility. It handles the bridge between the game and the hard drive. It turns the UserData class into a JSON string and writes it to user_data.json, and handles the logic for reading that file back in so the user stays logged in.
  • UserData: A simple data container. It holds the Username, Password, and a boolean for HasLoggedIn. This is the object that the SaveManager serializes to disk.
Summary of Changes
  • Decoupling: Moved away from the 2,000-line Java Client.java mess into a modular system where every feature has its own dedicated file.
  • State Control: Implemented a formal State Machine in Program.cs to handle transitions between the Menu, Login, and the (currently black) Server Selector.
  • Persistence: Added a JSON-based save system so the login process only happens once per user.
  • Visual Polish: Created a "hover-glow" effect for buttons and a self-healing background simulation that prevents the menu from ever looking "frozen."
