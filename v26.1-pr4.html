Bulletbox 26.1 Pre-Release 4
Pre-Release
With this update, we have rewritten the entire world save system to use SQLite instead of JSON files. This fixes a plethora of bugs. Along with this, we have added a few features for world saves. We have also fixed the implementation for version mismatch warnings. We have also compleely rewritten the biome generation.
World Saves
  • Rewrote the entire world save system to use SQLite instead of JSON files.
  • Added favorited worlds.
  • Fixed the implementation for version mismatch warnings.
New Textures

  • Updated the textures of resources/textures/ui/inventory/hotbar_active.png and resources/textures/ui/inventory/hotbar_deactive.png.
  • Added the following textures to resources/textures/ui/other/ favorited.png unfavorited.png.
Biomes and Features
  • Added the following new biomes: Mesa, Tundra, Frozen Ocea,n Icy Peaks, Swamp, Cherry Grove, and Rocky Beach.
  • Rewrote the biome generation to use Continentalness, Tempature, Humidity, Peaks, River, and Ashen perlin noises to generate biomes.
  • Biomes are now MUCH larger.
  • Mesa and Deserts now feature Cacti and Dead Bushes, Swamps are populated with Lilypads, Cherry Groves finally have their Cherry Trees, and the cold biomes (Tundra, Icy Peaks, Frozen Ocean) are now much more detailed with Frozen Trees, Berry Bushes, Ice Spikes, and varying Snow Piles.
The Biome Priority System
  • Biomes follows a strict hierarchy. If a chunk qualifies for multiple biomes, the code picks based on this order:
  • Sea Level: First, it checks continentalness for Oceans and Beaches.
  • Volcanic Override: Then it checks the ashen noise.
  • Hydrography: Then it carves the River.
  • Land Climate: Finally, if none of the above match, it uses peaks, temperature, and humidity to pick the specific land biome.
