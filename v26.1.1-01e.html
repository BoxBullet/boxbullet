Bulletbox 26.1.1-01e
Alpha Unreleased
Overhead Vitality System
  • World-Space Health Bars: Integrated a dynamic health bar system that renders directly above player entities, providing immediate visual feedback on player status during combat.
  • Percentage-Based Rendering: Implemented real-time calculation of health ratios to drive the foreground bar width, ensuring accuracy across different MaxHealth values.
  • High-Contrast Visuals: Adopted a layered UI approach with a black background and red vitality bar for maximum readability against the game world.
Server-Side Vitality Controller
  • Authoritative Health Management: Migrated all health calculations (Damage and Healing) to the server to ensure a secure, cheat-proof environment.
  • Bounds Enforcement: Implemented strict clamping logic within the Damage() and Heal() methods to prevent health values from exceeding maximums or dropping below zero.
  • Death & Respawn Logic: Created a server-side OnDeath hook that automatically manages player state resets and logs casualty events to the console.
Networking & State Synchronization
  • State-Sync Protocol (Packet ID 5): Developed a dedicated synchronization packet to push health and MaxHealth updates from the server to the client.
  • Event-Driven Updates: Configured the server to trigger a SyncHealth broadcast immediately following any health modification, ensuring low-latency updates for the HUD.
  • Remote Player Tracking: Enhanced the client-side Player class to store health data for all visible players, allowing the overhead bars to function for both the local user and adversaries.
UI & HUD Integration
  • Global State Sync: Updated the Connection listener to pipe incoming health packets directly into the PlayingState, enabling the main HUD to reflect the server's source of truth.
  • Post-Pixelation UI Layering: Ensured health bars and name tags are rendered after the pixelation filter pass to maintain high-definition text and UI clarity.
