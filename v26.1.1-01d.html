Bulletbox 26.1.1-01d
Alpha Unreleased
Dynamic Player Rendering
  • Procedural Nested Model: Replaced static image-based textures with a dynamic system of nested spinning rounded squares.
  • Dual-Axis Rotation: Implemented counter-clockwise rotation for the external Deep Green shape and clockwise rotation for the internal Magenta shape.
  • Shape Template System: Developed a "Texture Template" workflow to render high-quality rounded rectangles into a cache, enabling rotation of rounded corners without relying on the rlgl matrix stack.
  • Pixelation Filter: Integrated a 24x24 pixelation pass using RenderTexture2D with Point filtering to achieve a stylized, low-resolution aesthetic.
Camera & Matrix Optimization
  • Frame-Rate Independent Lerp: Upgraded the camera tracking logic using exponential decay math, ensuring smooth movement that feels consistent regardless of the user's FPS.
  • Matrix Conflict Resolution: Relocated the pixelation render pass from the Draw loop to the Update loop. This prevents BeginTextureMode from resetting the camera’s 2D transform, ensuring the player remains perfectly centered in the viewport.
  • Enhanced Centering: Increased LerpSpeed from 10.0 to 25.0 to provide tighter, more responsive camera tracking during high-velocity movement and knockback events.
Multiplayer & UI Improvements
  • Instance-Unique Rendering: Refactored static texture buffers into per-instance properties, ensuring that multiple players on screen can maintain unique rotation states and colors without visual ghosting.
  • UI Layering: Restored high-resolution name tags and health bars, drawing them after the pixelation pass to maintain readability over the stylized player model.
  • Pause Menu Connectivity: Implemented a functional "Disconnect" button in the PauseMenu to allow for clean session termination and network cleanup.
Code Cleanup & Refactoring
  • Texture Removal: Stripped all legacy texture-loading code related to player_idle and enemy_idle from Playing.cs and AssetManager.
  • Safe Null-Checking: Added null-conditional operators to the PauseMenu to prevent crashes when accessing the game state during transitions.
  • Namespace Optimization: Standardized the use of System.Numerics and Raylib_cs across all core game modules.
