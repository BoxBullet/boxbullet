Bulletbox 26.1.1-01g
Alpha Unreleased
Dedicated Launcher Framework
  • Unified Game Entry Point: Introduced a standalone Raylib-powered launcher responsible for client startup, update handling, and runtime management. (26.1.1-01g.1/1)
  • Markdown-Lite Patch Notes Renderer: Developed a lightweight formatted text engine supporting hierarchical headers (#, ##, ###) alongside automatic word wrapping for readable changelogs. (26.1.1-01g.1/2)
  • Scrollable Changelog Container: Implemented Scissor Mode clipping with smooth mouse-wheel scrolling and friction-based interpolation for polished navigation through patch notes. (26.1.1-01g.1/3)
  • Launcher Visual Identity: Designed a custom UI theme centered around the signature “Bulletbox Yellow” palette with cyan accent borders and dark-panel contrast. (26.1.1-01g.1/4)
Executable Discovery & Runtime Context
  • Dynamic Binary Resolution: Added an absolute-path discovery system capable of locating the BulletboxClient executable across varying project directory layouts. (26.1.1-01g.2/1)
  • Corrected Working Directory Handling: Resolved a major resource-loading issue where the client inherited the launcher’s directory context, preventing fonts and textures from loading properly. The launcher now explicitly sets the runtime working directory to the project root. (26.1.1-01g.2/2)
  • Automatic Launcher Shutdown: Configured the launcher process to terminate automatically after successfully handing execution over to the game client, reducing unnecessary resource usage. (26.1.1-01g.2/3)
UI Systems & Asset Reliability
  • Centralized Health Bar Assets: Refactored the HealthBar UI component to retrieve textures through the shared AssetManager, eliminating redundant LoadTexture operations. (26.1.1-01g.3/1)
  • Safe Resource Rendering: Added defensive checks throughout the UI rendering pipeline to prevent crashes caused by null texture references or temporarily missing asset paths. (26.1.1-01g.3/2)
  • Integrated Error Diagnostics: Implemented real-time launcher-side error reporting to surface missing binaries, failed launches, and build-related issues directly within the UI. (26.1.1-01g.3/3)
Performance & Portability Improvements
  • Wrapped Text Cache Optimization: Optimized launcher rendering performance by caching wrapped text calculations, preventing frame drops during rapid window resizing. (26.1.1-01g.4/1)
  • Portable Resource Discovery: Added upward directory traversal logic to automatically locate the resources/ directory relative to the executable location, improving portability across environments. (26.1.1-01g.4/2)
Development Notes
  • Ensure dotnet build has been executed within the BulletboxClient directory prior to launching the game.
  • The launcher currently defaults to connecting to 127.0.0.1:32308.
  • Future updates will include integrated local server startup directly from the launcher UI.
