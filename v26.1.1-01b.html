Bulletbox 26.1.1-01b
Alpha Unreleased
Networking Infrastructure
  • Client-Server Communication: Implemented a robust TCP-based architecture allowing for persistent multi-user connections and data streaming.
  • Packet-Based Protocol: Established a standardized communication layer using Packet IDs for discrete actions like movement, login, and combat.
  • Real-time Synchronization: Enabled position broadcasting so that all connected clients see other players moving in the world.
Combat System
  • Weapon Statistics Engine: Introduced a global library defining base damage, cooldowns, and range for Swords, Axes, Daggers, Spears, Scythes, and Kanabos.
  • Dynamic Math Formulas: Implemented non-linear scaling for combat logic, where damage and knockback are calculated based on the weapon's charge percentage.
  • Physics-Based Knockback: Replaced static coordinate updates with a smooth, velocity-based knockback system that uses friction decay for a natural physical feel.
  • Dagger Combo Mechanics: Added specialized logic for high-speed weapons, including exponential combo decay and attack speed penalties.
Inventory & Hotbar
  • Server-Authoritative Inventory: Implemented the ItemStack system where the server maintains the "Source of Truth" for all player items.
  • Hotbar Selection: Players can now select active weapon slots using number keys (1-6), with the server tracking the "Held Item" for combat validation.
  • Item Movement Logic: Added functionality to swap items between slots, synchronizing the changes across the network.
Health & State Management
  • Health Synchronization: Implemented a dedicated sync packet that updates the client HUD whenever damage or healing occurs on the server.
  • Death & Respawn Logic: Created basic server-side handling for player health reaching zero, including console logging and stat resets.
UI & Visual Enhancements
  • Visual HUD: Added a persistent health bar and a modular hotbar UI to the main gameplay screen.
  • Attack Charge Indicator: Implemented a dynamic UI bar that tracks weapon cooldowns and indicates the 35% minimum attack threshold.
  • Interactive Inventory Menu: Created a visual grid system for managing player items, including placeholder rendering and count displays.
  • Smooth Camera Lerp: Integrated the CameraManager to ensure the viewport follows the player smoothly even during rapid knockback events.
